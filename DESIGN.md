# GitSneaker

离线 Git 合并同步工具。把代码打包进U盘→人肉搬运→对端导入合并。

## 核心理念

```
A机器                         U盘                          B机器
main (有A1,A2)  ──bundle──▶  sync.bundle  ──pull──▶  main (合并A的改动)
```

本质是让两台网络不通的机器，像联网一样做正常的 git merge，完整保留提交历史和三方合并能力。

## 技术栈

- **前端**: 纯 HTML/CSS/JS，不引入框架
- **后端/打包**: Tauri 2（Rust 层只做 git 命令转发，不写业务逻辑）
- **Git 操作**: 直接调系统 git 命令（不依赖 libgit2）

## 核心流程

### 导出（Export）

用户在 A 机器上完成开发后：

1. 选择仓库
2. 自动检测上次同步点（通过本地 tag `sneaker-sync-*`）
3. 预览本次待导出的提交列表
4. 点击"导出"，生成 `sneaker-<timestamp>.bundle`
5. 打上同步标记 tag
6. 提示用户拷贝到 U 盘

### 导入（Import）

用户把 U 盘插到 B 机器：

1. 选择目标仓库
2. 选择 bundle 文件
3. 预览 bundle 中的提交列表
4. 点击"导入"，执行 `git pull <bundle>`
5. 自动判断：
   - 无冲突 → 提示"同步成功"
   - 有冲突 → 进入冲突解决视图

### 冲突解决（Merge）

三方合并视图，三个面板：

```
┌─────────────┬─────────────┬─────────────┐
│  LOCAL      │  BASE       │  REMOTE     │
│  (B的版本)  │  (共同祖先)  │  (A的版本)  │
├─────────────┴─────────────┴─────────────┤
│              合并结果（可编辑）           │
└─────────────────────────────────────────┘
```

- 冲突文件列表，点击切换
- 冲突块高亮，逐块选择保留哪边
- 支持手动编辑合并结果
- 完成后 `git add` + `git commit`

## 同步标记机制

导出时自动在当前 HEAD 打 tag：`sneaker-sync-<timestamp>`

下次导出时，以最近一个 `sneaker-sync-*` tag 为基准：

```bash
git bundle create output.bundle sneaker-sync-2026-07-01..HEAD
```

## 单文件便携

- 打包后只有一个 exe（Windows）/ app（macOS）
- 不写注册表，配置存仓库本地 `.sneaker.json`
- 可以直接放在 U 盘里运行
- 体积目标：< 10MB

## MVP 范围

**第一版做这些：**

- [ ] 仓库选择和最近记录
- [ ] 导出 bundle（自动基准+打tag）
- [ ] 导入 bundle（预览+执行）
- [ ] 冲突文件列表
- [ ] 三方合并视图（LOCAL/BASE/REMOTE）
- [ ] 冲突块逐一解决
- [ ] 完成合并提交

**不做：**

- 多仓库批量同步
- AI 自动解决冲突（二期考虑）
- 分支管理
- 远程仓库集成
